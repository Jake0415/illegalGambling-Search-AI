> 작성일: 2026-03-14

## 1. 서비스 개요

### 1.1 서비스명
**illegalGambling-Search-AI** (가칭: 도박사이트 자동 채증 시스템)

### 1.2 서비스 목표
인터넷상 불법 스포츠 도박 및 불법 경마 사이트를 자동으로 탐지하고, 법적 증거로 활용 가능한 3단계 스크린샷 채증을 자동 수행하는 통합 시스템

### 1.3 타겟 사용자
- 사행산업통합감독위원회
- 경찰청 사이버수사국
- 한국마사회 불법경마 감시팀
- 국민체육진흥공단
- 관련 수사 기관 및 감시 단체

### 1.4 핵심 가치
| 기존 방식 | 본 시스템 |
|-----------|----------|
| 수동 검색 및 신고 의존 | AI 기반 자동 탐지 |
| 수동 스크린샷 캡처 | 3단계 자동 채증 |
| 증거 무결성 관리 어려움 | 해시/타임스탬프 자동 관리 |
| 인력 집약적 (수사관 직접 수행) | 자동화로 대규모 병렬 처리 |
| 일회성 단속 | 지속적 모니터링 및 이력 관리 |

## 2. 핵심 기능 정의

> ⚠️ **실제 테스트 결과 반영** (2026-03-14, Playwright MCP 테스트)
> - Wego88, TotoHot 등 실제 사이트 접속 테스트 완료
> - 주요 장벽 우선순위: ① SMS OTP > ② 도메인 호핑 > ③ 팝업 처리 > ④ Cloudflare > ⑤ CAPTCHA

### 2.1 불법 사이트 자동 탐지 엔진
- **검색 기반 탐지**: Google/Bing API를 활용한 키워드 기반 불법 도박 사이트 검색
- **크롤링 기반 탐지**: 유해 사이트 네트워크 추적 (링크 분석, 광고 네트워크 추적)
- **AI 분류 모델**: 웹페이지 콘텐츠(HTML, 이미지, 텍스트)를 분석하여 불법 도박 사이트 여부 분류
- **URL 패턴 분석**: 도메인 특성, URL 구조 분석을 통한 의심 사이트 탐지
- **SNS/커뮤니티 모니터링**: 텔레그램, 트위터, 커뮤니티에서 불법 도박 홍보글 탐지
- **🔴 도메인 호핑 실시간 추적**: 불법 사이트는 빈번하게 도메인을 변경함 (테스트 시 6개 중 4개 접속 불가). 도메인 생존 상태를 주기적으로 체크하고, 신규 도메인을 실시간 추적하는 기능 필수

### 2.2 3단계 자동 채증 시스템

#### 1단계: 메인화면 채증
- 불법 사이트 접속 (Cloudflare/JS Challenge 자동 통과 — 테스트 확인됨)
- **팝업/광고 자동 닫기** (테스트 결과: 접속 시 3개 이상 팝업이 UI 차단)
- 메인 페이지 풀페이지 스크린샷
- HTML 소스 저장
- 네트워크 요청 로그 기록
- WHOIS/DNS 정보 수집

#### 2단계: 회원가입 및 배팅 시작 화면 채증
- **🔴 SMS OTP 인증 자동화 (최우선 과제)**:
  - 테스트 결과 Wego88 등 대부분의 사이트에서 CAPTCHA 없이 SMS OTP만 요구
  - +82 한국 가상번호 발급 → 폼에 입력 → OTP 수신 → 인증코드 자동 입력
  - GrizzlySMS/PVAPins 등 다중 서비스 풀링
- **CAPTCHA 하이브리드 3단계 통과** (일부 사이트에서 출현 시):
  1. CapSolver API 자동 풀이 (AI, 3~9초)
  2. 2Captcha 폴백 (사람 풀이, 10~60초)
  3. 대시보드 수동 개입 (CDP "Pause & Attach" 패턴)
- 회원가입 완료 화면 캡처
- 배팅 메뉴/시작 화면 캡처 (테스트 결과: 로그인 없이도 배팅 목록 접근 가능한 사이트 존재)
- 입금/충전 안내 화면 캡처

#### 실제 테스트 기반 — 회원가입 폼 구조 (Wego88 사례)

| 필드 | 유형 | 필수 | 자동화 방법 |
|------|------|------|-----------|
| 회원아이디 | 텍스트 | ✅ | 랜덤 생성 (영문+숫자) |
| 비밀번호 | 텍스트 | ✅ | 자동 생성 (4~20자, 문자+숫자) |
| 비밀번호 확인 | 텍스트 | ✅ | 동일값 |
| 통화 | 드롭다운 | ✅ | KRW 기본값 유지 |
| 전화번호 | +82 | ✅ | **가상번호 서비스 API** |
| OTP 인증코드 | 텍스트 | ✅ | **SMS 수신 → 파싱 → 자동 입력** |
| 친구 추천 | 텍스트 | ❌ | 비워둠 |

#### 3단계: 실제 배팅 화면 채증
- 배팅 가능 종목/경기 목록 캡처
- 배팅 폼/배당률 화면 캡처
- 배팅 실행 화면 캡처 (최소 금액)
- 배팅 확인/결과 화면 캡처
- ※ 배팅 페이지에서 간헐적 CAPTCHA 출현 시 동일 하이브리드 전략 적용

### 2.3 팝업/광고 자동 처리 시스템
> 실제 테스트에서 확인된 필수 기능

- 접속 시 프로모션 팝업 자동 감지 및 닫기
- `dialog` 오버레이가 클릭을 차단하는 경우 → 닫기 버튼 자동 탐색·클릭
- iframe 내 광고 팝업 처리
- 팝업 닫기 후 원래 작업 자동 재개
- 사이트별 팝업 패턴 학습 (어댑터 패턴)

### 2.4 CAPTCHA/OTP 수동 개입 시스템 (대시보드)
- **`/investigations/captcha-queue`**: CAPTCHA + OTP 수동 개입 대기 큐
- CDP(Chrome DevTools Protocol) WebSocket으로 원격 브라우저 화면 스트리밍
- 사용자가 대시보드에서 직접 CAPTCHA 풀이 또는 OTP 입력
- 풀이 완료 감지 시 자동화 자동 재개
- Slack + 웹 푸시로 대기 알림 (모바일 대응)
- 상세: `docs/planning/captcha-strategy.md` 참조

### 2.5 디지털 증거 관리 시스템
- **해시값 생성**: 모든 캡처 파일에 SHA-256 해시값 자동 생성
- **타임스탬프**: RFC 3161 표준 타임스탬프 부여 (TSA 서버 연동)
- **메타데이터 기록**: 캡처 시점, 접속 IP, 프록시 정보, 브라우저 정보 등
- **체인 오브 커스터디**: 증거 수집부터 제출까지 전 과정 감사 로그
- **증거 패키지 생성**: 법원 제출용 증거 패키지 자동 생성 (PDF 보고서 포함)

### 2.6 대시보드 및 관리 화면
- 탐지된 불법 사이트 목록 및 상태 관리
- 채증 진행 상황 실시간 모니터링
- 채증 결과 조회 및 검색
- 통계 및 리포트 (유형별, 기간별, 상태별)
- 사용자/권한 관리

### 4.4 AI/ML (탐지) — LangChain/LangGraph 기반 (2026-03-15 결정)
| 구분 | 기술 | 비고 |
|------|------|------|
| 사이트 분류 | **LangChain + Claude Haiku 4.5** | 도박 여부 판정, F1 94-95%, few-shot |
| 채증 워크플로우 | **LangGraph 상태 머신** | 3단계 파이프라인, 분기/재시도/수동 개입 오케스트레이션 |

| 보고서 생성 | **LangChain + Claude Sonnet 4.6** | 한국어 수사 보고서 자동 생성 |
| CAPTCHA 판단 | **LangGraph 분기 로직** | 유형 판별 -> API 풀이/수동 개입/스킵 분기 |
| 향후 확장 | XGBoost/BERT 하이브리드 | 데이터 축적 후 점진 전환 (Phase 4 이후) |

## 5. 외부 서비스/API 연동 계획

| 서비스 | 용도 | API 형태 | 예상 비용 (월) | 우선순위 |
|--------|------|---------|--------------|---------|
| **GrizzlySMS / PVAPins** | **SMS OTP 인증** | REST API | **~$100-300** | **1순위** |
| **IPRoyal / SOAX** | **프록시 (도메인 호핑 대응)** | HTTP Proxy | **~$100-500** | **2순위** |
| Google Custom Search API | 불법 사이트 검색 | REST API | ~$50 (10,000쿼리) | 3순위 |
| CapSolver | CAPTCHA 풀기 | REST API | ~$2 (실제 출현 빈도 낮음) | 4순위 |
| 2Captcha | CAPTCHA 폴백 | REST API | ~$1 (10% 폴백) | 5순위 |
| FreeTSA.org | 타임스탬프 | RFC 3161 | 무료 | - |
| Similarweb API | 트래픽 분석 (선택) | REST API | $200+ | 선택 |

> ⚠️ **테스트 결과 반영**: CAPTCHA 비용은 예상보다 훨씬 낮음 (월 ~$2). 실제 비용 중심은 **SMS 인증($100~300)**과 **프록시($100~500)**.

**예상 월 운영비**: 외부 서비스만 $450~$1,050 (인프라 제외, CAPTCHA 비용 대폭 하향)

## 6. MVP 범위 정의

### Phase 1: MVP (4-6주)
**목표**: 핵심 채증 파이프라인 동작 검증

| 기능 | 세부 내용 | 우선순위 |
|------|----------|---------|
| 수동 URL 입력 | 관리자가 URL을 직접 입력 | P0 |
| 1단계 채증 | 메인화면 스크린샷 + HTML 저장 | P0 |
| 증거 해시 생성 | SHA-256 해시값 자동 생성 | P0 |
| 기본 대시보드 | 사이트 목록, 채증 상태 조회 | P0 |
| 기본 인증 | 관리자 로그인 | P0 |

### Phase 2: 자동 채증 (4-6주)
**목표**: 2, 3단계 채증 자동화

| 기능 | 세부 내용 | 우선순위 |
|------|----------|---------|
| **SMS 인증 자동화** | **가상번호 서비스 연동 (최우선)** | **P1** |
| **팝업/광고 자동 처리** | **접속 시 팝업 자동 닫기** | **P1** |
| 2단계 채증 | 회원가입 + 배팅 시작 화면 | P1 |
| 3단계 채증 | 실제 배팅 화면 캡처 | P1 |
| 프록시 연동 | 로테이션 프록시 적용 | P1 |
| CAPTCHA 솔버 연동 | CapSolver API 연동 (출현 시만) | P1 |
| 타임스탬프 | RFC 3161 타임스탬프 연동 | P1 |

### Phase 3: AI 탐지 (6-8주)
**목표**: 자동 탐지 파이프라인 구축

| 기능 | 세부 내용 | 우선순위 |
|------|----------|---------|
| 키워드 검색 크롤러 | Google API 기반 자동 검색 | P2 |
| AI 분류 모델 | 도박 사이트 판별 ML 모델 | P2 |
| 스케줄 실행 | 주기적 자동 탐지 | P2 |
| 알림 시스템 | 신규 탐지 시 알림 | P2 |

### Phase 4: 고도화 (4-6주)
**목표**: 운영 안정성 및 고급 기능

| 기능 | 세부 내용 | 우선순위 |
|------|----------|---------|
| 증거 패키지 생성 | 법원 제출용 PDF 보고서 | P3 |
| 통계/분석 대시보드 | 유형별, 기간별 통계 | P3 |
| SNS 모니터링 | 텔레그램/트위터 감시 | P3 |
| 사이트 네트워크 분석 | 운영자 그룹핑, 연결망 분석 | P3 |
