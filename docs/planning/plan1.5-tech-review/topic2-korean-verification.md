## 1. SMS OTP 가상번호 서비스

### 1.1 서비스별 비교 표

| 서비스 | +82 제공 | 가격/건 (USD) | API 제공 | Non-VoIP | 비고 |
|--------|---------|-------------|---------|---------|------|
| **GrizzlySMS** | O (203개 재고 확인) | ~$0.86 | O (REST API) | 혼합 | Python 라이브러리 존재, USD 전환 완료 |
| **SMS-Activate** | O (광고 중) | ~$0.50-1.50 (추정) | O (REST API v2) | 혼합 | 가장 큰 글로벌 서비스, API 문서 충실 |
| **5sim** | O (광고 중) | $0.014~ (최저가, 실제 KR은 더 높음) | O (JWT 기반 REST API) | 혼합 | 180개국 지원, 자동화 특화 |
| **PVAPins** | O | $0.10~ (앱/국가 조합별 상이) | O (API + Webhook) | **O (실제 통신사 번호)** | 프리미엄 Non-VoIP 라우트 제공 |
| **SMSOnline.cloud** | O (무료) | 무료 | X (웹 전용) | X (VoIP/공유) | KakaoTalk, Naver 등 지원 표시 |
| **Quackr** | O (무료) | 무료 | X (웹 전용) | X (VoIP/공유) | 실시간 메시지 표시, 클라우드 기반 |

### 1.2 +82 실제 제공 여부 및 성공률

**실재고 확인된 서비스:**
- **GrizzlySMS**: 검색 시점 203개 한국 번호 재고 확인. +82 10 형식(010-xxxx-xxxx)의 모바일 번호 제공.
- **PVAPins**: 한국 번호 제공 확인. Non-VoIP 실제 통신사 번호를 사용하므로 엄격한 플랫폼에서도 OTP 수신 가능성이 높음.
- **SMSOnline.cloud / Quackr**: 무료 한국 번호 제공하나, 공유 번호이므로 재사용이 빠르고 도박사이트에서 차단될 가능성이 높음.

**핵심 리스크:**
- 도박사이트들은 가상번호/VoIP 번호를 탐지하여 차단하는 경우가 있음
- 무료 공유 번호는 이미 블랙리스트에 등재되어 있을 가능성이 높음
- **Non-VoIP(실제 통신사 SIM 기반) 번호가 필수적**일 가능성이 높음

**예상 성공률:**
- Non-VoIP 프리미엄 번호(PVAPins 등): 60-80%
- 일반 VoIP 가상번호(GrizzlySMS, 5sim 등): 30-50%
- 무료 공유 번호(SMSOnline, Quackr): 10-20% (거의 불가)

### 1.3 API 연동 평가

**1순위: SMS-Activate (sms-activate.io)**
- REST API v2 제공 (`/api2` 엔드포인트)
- 번호 요청 → SMS 대기 → 코드 수신의 풀 사이클 API 지원
- 다양한 언어 SDK 존재
- 문서화 수준: 우수

**2순위: GrizzlySMS**
- REST API + Python 라이브러리 (GitHub: giwih/grizzlysms)
- `maxPrice` 파라미터로 최대 지불 가격 설정 가능
- `getPrices` → `getNumber` → `getStatus` 플로우
- API 키 기반 인증, 문서: https://grizzlysms.com/docs

**3순위: 5sim**
- JWT 기반 Bearer 토큰 인증
- REST API v1 (`docs.5sim.net/v1/en/`)
- 번호 구매, 상태 확인, 취소 등 전체 라이프사이클 API
- Python 클라이언트 존재 (GitHub: ErikPelli/fivesim)

**4순위: PVAPins**
- API + Webhook 지원
- 자동 재시도/백오프 로직 내장 가능
- RPA/봇 연동에 최적화
- Non-VoIP 라우트라는 점에서 실질적 성공률 면에서 1순위

## 2. PASS 앱 인증

### 2.1 기술 플로우

PASS는 SKT, KT, LG U+ 3사가 공동 제공하는 본인확인 서비스이다.

**인증 프로세스:**
1. 웹사이트에서 PASS 인증 요청
2. 사용자 입력: 이름, 생년월일, 통신사, 휴대폰 번호
3. PASS 앱으로 **Push 알림** 전송 (Server-to-Server, KISA 권고 방식)
4. 앱에서 인증 요청 정보 확인
5. **생체인증(지문/얼굴)** 또는 **6자리 PIN** 입력
6. 전자서명 생성 → 인증 완료

**핵심 특징:**
- 본인 명의의 실제 통신사 가입 휴대폰이 필수
- SIM 카드와 USIM이 바인딩되어 있어 기기 변경 시 재등록 필요
- 번호 변경 시 기존 인증서 무효화

### 2.2 자동화 가능성 평가

**ADB/Appium 자동화 시나리오:**
- Appium UIAutomator2 드라이버로 Android PASS 앱 UI 조작 가능
- Push 알림 수신 → 앱 열기 → PIN 입력 자동화는 기술적으로 가능
- 생체인증은 Android 에뮬레이터의 Fingerprint API mock으로 우회 가능

**실질적 한계:**
- **실제 통신사 가입 SIM이 필수**: 가상번호로는 PASS 앱 등록 자체가 불가능
- 대량(50+) 자동화 시 실제 SIM 카드 50개 + 단말 50대 필요
- PASS 앱의 루팅/에뮬레이터 탐지 기능이 있을 수 있음
- **결론: 대량 자동화에 비현실적**

### 2.3 도박사이트 PASS 인증 사용 빈도

- 불법 도박사이트에서 PASS 인증을 사용하는 경우는 **드물 것으로 추정**
- PASS 연동은 정식 사업자 계약이 필요하므로, 불법 사이트가 정식 연동할 수 없음
- 다만, 일부 사이트가 자체적으로 PASS와 유사한 UI를 모방하여 개인정보를 수집하는 경우 존재
- **대부분의 불법 도박사이트는 SMS OTP 방식을 채택**

## 3. 통신사 인증 (SKT/KT/LGU+)

### 3.1 SMS OTP와의 차이점

| 구분 | SMS OTP | 통신사 인증 |
|------|---------|-----------|
| 인증 방식 | SMS로 OTP 코드 수신 후 입력 | 통신사 서버에서 가입자 정보 직접 대조 |
| 필요 정보 | 전화번호만 | 이름 + 생년월일 + 통신사 + 전화번호 |
| 본인확인 수준 | 번호 소유 확인 | 실명 + 번호 소유 확인 |
| 가상번호 사용 | 가능 (VoIP 탐지 시 실패) | **불가** (통신사 가입자 DB 대조) |
| 기술 연동 | 단순 (SMS 수신만) | 통신사 API 연동 필요 |

### 3.2 가상번호의 통신사 인증 통과 가능성

- **불가능**: 통신사 인증은 SKT/KT/LGU+의 가입자 DB와 직접 대조
- KT TwoPhone(4,400원/월) 등 통신사 부가 서비스 가상번호도 본인인증 **미지원**
- SKT Number Plus 2(3,850원/월) 역시 마찬가지
- 해외 SMS 서비스(GrizzlySMS 등)의 번호는 한국 통신사 가입자 DB에 존재하지 않으므로 완전히 불가

### 3.3 Playwright 처리

- 통신사 인증은 보통 **팝업 윈도우**로 열림
- Playwright에서 `page.waitForEvent('popup')` 으로 새 창 캡처 가능
- 팝업 내 iframe 처리가 필요할 수 있음
- 그러나 **가상번호로 통과 자체가 불가능**하므로 Playwright 처리는 의미 없음

### 3.4 결론

통신사 인증을 요구하는 도박사이트는 가상번호 기반 자동화로 우회 불가. 단, **대부분의 불법 도박사이트는 정식 통신사 인증 API를 사용하지 않고 단순 SMS OTP를 사용**할 가능성이 높음 (정식 통신사 API 계약 불가하므로).

## 4. 아이핀(i-PIN) 인증

### 4.1 개요

i-PIN은 주민등록번호 대체 수단으로 정부가 주도하는 온라인 본인확인 서비스이다.

**발급 기관:**
- NICE평가정보 (niceipin.co.kr)
- 코리아크레딧뷰로 (KCB)
- 공공아이핀 (gpin.go.kr)

**발급 요건:**
- 온라인: 공동인증서 + 본인 명의 휴대폰 (SMS 인증 필요)
- 오프라인: 행정복지센터 방문 (신분증 필요)

### 4.2 프로그래밍 자동화 가능성

- i-PIN 발급 자체가 **본인 명의 휴대폰 SMS 인증** 또는 **공동인증서**를 요구
- 순환 의존성 문제: i-PIN 발급에 SMS 인증이 필요 → SMS 인증이 문제라면 i-PIN 발급도 불가
- i-PIN 연동은 NICE, KCB 등과 **사업자 계약** 후 API 사용 (개인 사용 불가)
- eGovFrame에서 공공 i-PIN 실명인증 모듈이 존재하나, 사업자용

### 4.3 도박사이트 수용률

- 불법 도박사이트에서 i-PIN 인증을 지원할 가능성: **매우 낮음**
- i-PIN 연동은 NICE/KCB와 정식 사업자 계약 필요 → 불법 사이트는 불가
- i-PIN은 주로 공공기관, 포털, 게임 사이트 등 합법적 서비스에서 사용
- 나무위키에 따르면 i-PIN 자체의 사용률도 지속적으로 감소 추세

### 4.4 법적 함의

- 수사 목적이라 해도 타인 명의 i-PIN 발급은 **명백한 불법** (개인정보보호법 위반)
- 수사관 본인 명의로 i-PIN 발급 후 도박사이트 가입은 가능하나, 사이트가 i-PIN을 지원해야 함
- **실용성 없음**: 도박사이트가 i-PIN을 지원하지 않으므로 논의 자체가 불필요

## 5. 카카오/네이버 간편인증

### 5.1 카카오 인증 (카카오싱크)

**기술 플로우:**
1. 사이트에서 카카오 로그인/인증 요청
2. 카카오 OAuth 페이지로 리다이렉트
3. 카카오 계정 로그인 (ID/PW)
4. 동의 화면에서 정보 제공 동의
5. 인증 완료 + 사용자 정보 전달

**일회용 계정 생성:**
- 카카오 계정 생성 시 **본인 명의 휴대폰 번호 인증** 필수
- 순환 의존성: 카카오 계정 → 휴대폰 인증 → 다시 SMS OTP 문제
- 이메일만으로 계정 생성 불가 (한국 서비스 기준)

**인증서 발급 조건:**
- 카카오 인증서: 본인 명의 휴대폰 + 1원 계좌 인증 필요
- 폰 변경/번호 변경 시 재발급 필요

### 5.2 네이버 인증

- 네이버 계정 역시 **휴대폰 번호 인증** 필요
- 네이버 인증서: 본인 명의 휴대폰 + 신분증 촬영 필요
- 순환 의존성 동일

### 5.3 도박사이트 적용 가능성

- 불법 도박사이트가 카카오/네이버 OAuth를 정식 연동할 가능성: **없음**
- 카카오/네이버 비즈니스 파트너 계약이 필요하며, 불법 사업자는 승인 불가
- 일부 사이트가 "카카오톡으로 문의" 등은 제공하나, 이는 인증이 아닌 고객 지원

### 5.4 결론

카카오/네이버 간편인증은 **순환 의존성**(계정 생성에 휴대폰 인증 필요)과 **불법 사이트의 미지원**으로 인해 실용적이지 않음.

## 6. ARS 음성인증

### 6.1 개요

ARS(Automatic Response System) 인증은 자동 음성 전화를 통해 본인확인하는 방식이다.

**일반적 플로우:**
1. 사이트에서 ARS 인증 요청
2. 등록된 전화번호로 자동 음성 전화 발신
3. 화면에 표시된 4자리 OTP를 전화기에 입력 (또는 반대)
4. 인증 완료

**사용 사례:** 주로 은행, 카드사, 보험사 등 금융기관에서 사용 (예: 하나카드 ARS 인증)

### 6.2 가상번호 음성 통화 수신 가능성

**한국 VoIP 번호(070):**
- 음성 통화 수신 가능
- 한국 내 로컬 프레즌스 필요
- 제공 업체: AVOXI ($6/월~), CallHippo ($1/월~), CloudTalk, Freezvon
- 070 번호는 VoIP임이 명확하므로 본인인증용으로는 거부될 가능성 높음

**해외 SMS 서비스의 음성 통화 지원:**
- GrizzlySMS, 5sim, SMS-Activate 등은 **SMS 전용** (음성 통화 미지원)
- PVAPins의 Non-VoIP 번호는 SMS 전용 (음성 미지원)

### 6.3 프로그래밍 처리

- ARS 전화 수신 + DTMF 톤 입력은 VoIP/SIP 클라이언트로 자동화 가능
- Twilio, Nexmo 등의 프로그래머블 보이스 API 활용 가능
- 그러나 한국 번호 확보 문제는 동일하게 존재

### 6.4 도박사이트 ARS 인증 사용 빈도

- **거의 없음**: ARS 인증은 정식 금융기관 전용
- 불법 도박사이트는 ARS 인증 인프라를 구축하지 않음
- **검토 불필요**

| **통신비밀보호법** | 제13조 | 범죄수사를 위한 통신사실 확인자료 제공 (법원 허가 필요) |
| **정보통신망법** | - | 정보통신망 이용 관련 규제 |
| **국민체육진흥법** | - | 불법 도박 개장/이용 처벌 근거 |

### 7.2 위장수사의 합법성

**함정수사 판례 기준:**
- **기회제공형 함정수사**: 이미 범의를 가진 자에게 범행 기회 제공 → **원칙적 허용**
- **범의유발형 함정수사**: 범의 없는 자를 유도 → **위법**

**불법 도박사이트 위장가입의 경우:**
- 이미 운영 중인 불법 도박사이트에 증거 수집 목적으로 가입하는 것은 기회제공형에 해당할 가능성
- 그러나 **불법 도박사이트 위장가입에 대한 직접적 판례는 확인되지 않음**
- 참고: 아동 성착취물 등 특정 범죄에 대해서는 신분위장수사가 명시적으로 허용됨

### 7.3 가상번호 사용의 법적 리스크

**전기통신사업법 관점:**
- 거짓 표시 전화번호 사용에 관한 고시 존재
- 수사 목적이라도 통신사 약관 위반 가능성
- 해외 SMS 서비스 이용 자체는 불법이 아니나, 타인 명의 도용은 불법

**권장 법적 절차:**
1. 수사 개시 결정 및 내부 승인
2. 필요시 법원 영장 발부 (통신사실확인자료 등)
3. 수사관 본인 명의 또는 수사용 번호 사용
4. 모든 활동 기록 보존 (증거능력 확보)

### 7.4 헌법재판소 결정 관련

- 2022년 헌법재판소는 전기통신사업법 제83조 제3항의 수사기관 통신자료 제공 요청 부분에 대해 **헌법불합치** 결정
- 법원 허가 없는 통신자료 제공의 위헌성 지적
- 수사기관의 통신 관련 정보 수집에 대한 법적 요건 강화 추세

## 8. 최종 추천 (우선순위 체인)

### 8.1 우선순위 체인

```
Primary     → SMS OTP 가상번호 (Non-VoIP 우선)
Fallback 1  → SMS OTP 가상번호 (일반 VoIP)
Fallback 2  → 수사용 실제 선불 SIM 카드
Manual      → 수동 개입 (사람이 직접 인증)
```

### 8.2 상세 추천

#### Primary: PVAPins Non-VoIP 번호 + API 자동화

- **이유**: 실제 통신사 SIM 기반 번호로 VoIP 탐지 우회 가능성 높음
- **비용**: ~$0.50-2.00/건, 월 500건 기준 $250-1,000
- **API 연동**: Webhook 기반 자동화 가능, RPA/봇 최적화
- **리스크**: 한국 번호 재고 변동, 도박사이트별 차단 여부 불확실
- **구현**: Playwright 가입 플로우 + PVAPins API 연동

#### Fallback 1: GrizzlySMS 또는 SMS-Activate + API 자동화

- **이유**: 대량 재고 확보 가능 (GrizzlySMS 203개 확인), API 문서 우수
- **비용**: ~$0.86/건 (GrizzlySMS), 월 500건 기준 ~$430
- **API 연동**: REST API, Python SDK 존재
- **리스크**: VoIP 번호일 경우 도박사이트 차단 가능 (성공률 30-50%)
- **구현**: Primary 실패 시 자동 전환

#### Fallback 2: 수사용 선불 SIM 카드 풀

- **이유**: 실제 통신사 번호이므로 모든 인증 통과 가능
- **비용**: 선불 SIM 약 5,000-10,000원/개 + 월 유지비
- **방법**: Android 단말 + ADB/Appium으로 SMS 자동 읽기
- **한계**: 50개 이상 확보/관리의 물리적 부담, 법적 절차 필요
- **구현**: SMS Forwarder 앱 또는 ADB `dumpsys` 명령으로 SMS 읽기 자동화

#### Manual: 수동 개입 프로토콜

- **트리거**: 위 모든 자동화 실패 시
- **방법**: Slack/Teams 알림 → 담당자가 직접 SMS 인증 수행
- **목표 응답 시간**: 5분 이내
- **구현**: Playwright `page.pause()` 또는 WebSocket 기반 대기 로직

### 8.3 구현 아키텍처

```
┌─────────────────────────────────────────────────┐
│              Playwright 자동 가입 엔진            │
│                                                  │
│  1. 사이트 접속 → 가입폼 작성                      │
│  2. 전화번호 입력 (SMS 서비스에서 번호 할당)         │
│  3. OTP 요청 버튼 클릭                             │
│  4. SMS 서비스 API로 OTP 코드 폴링 (최대 120초)     │
│  5. OTP 코드 입력 → 가입 완료                      │
│                                                  │
│  ┌──────────┐  실패  ┌──────────┐  실패  ┌──────┐ │
│  │ PVAPins  │──────→│GrizzlySMS│──────→│ SIM  │ │
│  │(Non-VoIP)│       │ (VoIP)   │       │ Pool │ │
│  └──────────┘       └──────────┘       └──────┘ │
│        │                 │                 │     │
│        └───── 모두 실패 ──┴────── 수동 개입 ─┘     │
└─────────────────────────────────────────────────┘
```

### 8.4 50건 동시 처리 가능성 평가

| 방법 | 50건 동시 가능 | 제약사항 |
|------|-------------|---------|
| PVAPins | 가능 (API 기반) | 한국 번호 재고에 따라 변동 |
| GrizzlySMS | 가능 (203개 재고) | 동시 요청 시 재고 소진 빠름 |
| SMS-Activate | 가능 (대규모 인프라) | 한국 번호 실재고 확인 필요 |
| 5sim | 가능 (API 기반) | 한국 번호 가용성 불확실 |
| 선불 SIM | 비현실적 | 물리적 단말 50대 필요 |

### 8.5 즉시 실행 항목

1. **PVAPins 테스트 계정 생성** → 한국 번호 1-2개 구매 → 실제 도박사이트 OTP 수신 테스트
2. **GrizzlySMS 테스트** → $5 충전 → 한국 번호 할당 → OTP 수신 확인
3. **실패 시 SMS-Activate, 5sim 순차 테스트**
4. **성공한 서비스의 API 연동 PoC 구현** (Node.js/TypeScript)
5. **법적 검토 요청** → 수사기관 법무팀에 가상번호 사용 적법성 확인
