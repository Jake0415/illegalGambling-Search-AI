# Task 021: 반응형 디자인 + 다크 모드 + 로딩/에러 처리 가이드

## 문서 목적

전체 페이지에 걸쳐 일관되게 적용해야 할 반응형 레이아웃, 다크 모드, 로딩 상태, 에러 처리 패턴을 정의한다. 개별 페이지 화면 정의서가 아닌 **공통 디자인 가이드**이다.

---

## 1. 반응형 레이아웃 (FR-UI-033)

### 브레이크포인트 정의

| 뷰포트 | 너비 | TailwindCSS | 레이아웃 |
|--------|------|-------------|----------|
| 모바일 | < 640px | (기본) | 하단 네비게이션 바, 카드 1열, 풀스크린 모달 |
| 태블릿 | 640~1023px | `sm:`, `md:` | 사이드바 오버레이(Sheet), 카드/테이블 2열 |
| 데스크톱 | >= 1024px | `lg:`, `xl:` | 좌측 사이드바(240px) + 메인 콘텐츠 |

### 사이드바 동작

```
데스크톱 (lg+):
+----------+------------------------------------------+
| Sidebar  | Main Content                             |
| (240px)  | (flex-1)                                 |
| 고정     |                                          |
| 접기 토글|                                          |
+----------+------------------------------------------+

태블릿 (md):
+--------------------------------------------------+
| [☰] Header                                       |
+--------------------------------------------------+
| Main Content (full width)                         |
|                                                   |
| (사이드바: Sheet 오버레이, 햄버거 메뉴 클릭 시)   |
+--------------------------------------------------+

모바일 (sm):
+--------------------------------------------------+
| Header (간소화)                                    |
+--------------------------------------------------+
| Main Content (full width)                         |
| 카드 기반 리스트 (테이블 대체)                    |
+--------------------------------------------------+
| [홈] [사이트] [채증] [증거] [더보기]              |
| 하단 네비게이션 바 (고정)                         |
+--------------------------------------------------+
```

### 테이블 반응형 전략

| 뷰포트 | 전략 |
|--------|------|
| 데스크톱 | 모든 컬럼 표시, 수평 스크롤 없음 |
| 태블릿 | 주요 컬럼만 표시 (3~5개), 행 클릭 시 확장 행으로 나머지 표시 |
| 모바일 | DataTable 대신 카드 리스트로 전환 (각 항목이 Card 컴포넌트) |

### 터치 인터랙션

| 항목 | 규격 |
|------|------|
| 터치 타겟 | 최소 44x44px |
| 스와이프 | 사이드바 열기/닫기 (좌→우 / 우→좌) |
| 탭 앤 홀드 | 테이블 행 컨텍스트 메뉴 (모바일) |

### 컴포넌트 매핑

| 컴포넌트 | 역할 |
|----------|------|
| `frontend/src/components/layout/sidebar.tsx` | 사이드바 (데스크톱: 고정, 모바일: Sheet) |
| `frontend/src/components/layout/dashboard-header.tsx` | 헤더 (햄버거 메뉴, 알림 벨, 사용자 메뉴) |
| shadcn/ui `Sheet` | 모바일 사이드바 오버레이 |
| shadcn/ui `NavigationMenu` | 하단 네비게이션 (모바일) |

## 2. 다크 모드 (FR-UI-034)

### 모드 전환 방식

| 모드 | 설명 | 구현 |
|------|------|------|
| 시스템 | OS 설정 따르기 (기본값) | `prefers-color-scheme` |
| 라이트 | 강제 라이트 모드 | `next-themes` class 전략 |
| 다크 | 강제 다크 모드 | `next-themes` class 전략 |

### 전환 UI 위치

- 데스크톱: 사이드바 하단 또는 헤더 우측
- 모바일: 설정 메뉴 내

### 색상 토큰 매핑 (TailwindCSS v4 + shadcn/ui)

| 요소 | 라이트 모드 | 다크 모드 |
|------|------------|-----------|
| 배경 (page) | `bg-background` (white) | `bg-background` (zinc-950) |
| 카드 배경 | `bg-card` (white) | `bg-card` (zinc-900) |
| 텍스트 (주) | `text-foreground` (zinc-950) | `text-foreground` (zinc-50) |
| 텍스트 (보조) | `text-muted-foreground` (zinc-500) | `text-muted-foreground` (zinc-400) |
| 테이블 행 (짝수) | `bg-muted/50` | `bg-muted/50` |
| 상태 배지 (green) | `bg-emerald-100 text-emerald-800` | `bg-emerald-900/30 text-emerald-400` |
| 상태 배지 (red) | `bg-red-100 text-red-800` | `bg-red-900/30 text-red-400` |
| 차트 색상 | Recharts 기본 팔레트 | 다크 팔레트 (밝은 톤) |

### 접근성 대비비 (WCAG 2.1 AA)

| 항목 | 기준 |
|------|------|
| 일반 텍스트 | 대비비 4.5:1 이상 |
| 대형 텍스트 (18px+) | 대비비 3:1 이상 |
| UI 컴포넌트/아이콘 | 대비비 3:1 이상 |

### 사용자 설정 저장

- `localStorage`에 `theme` 키로 저장 (`'light'` / `'dark'` / `'system'`)
- 새로고침/세션 간 유지
- `next-themes` `ThemeProvider`로 관리

## 3. 로딩 상태 처리 (FR-UI-035)

### 페이지 전환 로딩

| 상태 | UI |
|------|-----|
| 페이지 전환 시작 | 상단 프로그레스 바 (NProgress 스타일) |
| 서버 컴포넌트 로딩 | Next.js `loading.tsx` → `PageSkeleton` 표시 |

### 데이터 로딩

| 상태 | UI | 컴포넌트 |
|------|-----|----------|
| 테이블 데이터 로딩 | 테이블 스켈레톤 (5행 x N열) | `TableSkeleton` |
| 카드 데이터 로딩 | 카드 스켈레톤 (헤더+본문 줄) | `CardSkeleton` |
| 전체 페이지 로딩 | 페이지 스켈레톤 (헤더+테이블) | `PageSkeleton` |
| 차트 데이터 로딩 | 차트 영역 스켈레톤 | `Skeleton` (커스텀 크기) |
| 버튼 클릭 후 | 로딩 스피너 + 버튼 비활성화 | `Button disabled + Loader2 아이콘` |

### 파일별 매핑

| 파일 | 용도 |
|------|------|
| `frontend/src/app/loading.tsx` | 전역 로딩 (PageSkeleton) |
| 각 라우트 `loading.tsx` | 라우트별 로딩 |
| `frontend/src/components/common/loading-skeleton.tsx` | TableSkeleton, CardSkeleton, PageSkeleton |

## 4. 에러 처리 (FR-UI-035)

### React Error Boundary

| 레벨 | 파일 | 동작 |
|------|------|------|
| 루트 | `frontend/src/app/error.tsx` | 전역 에러 폴백 ("문제가 발생했습니다" + 재시도 버튼) |
| 라우트별 | 각 `error.tsx` | 섹션 단위 에러 격리 |

### API 에러 처리

| HTTP 상태 | UI |
|-----------|-----|
| 400 Bad Request | 인라인 `FormMessage` (폼 필드별) |
| 401 Unauthorized | `/login`으로 리디렉트 + toast("세션이 만료되었습니다") |
| 403 Forbidden | toast.error("접근 권한이 없습니다") |
| 404 Not Found | `frontend/src/app/not-found.tsx` (커스텀 404) |
| 500 Internal Server Error | toast.error("서버 오류가 발생했습니다") + 재시도 버튼 |

### 404 Not Found 페이지

```
+--------------------------------------------------+
|                                                    |
|              404                                   |
|              페이지를 찾을 수 없습니다             |
|                                                    |
|   요청하신 페이지가 존재하지 않거나                |
|   이동되었을 수 있습니다.                          |
|                                                    |
|              [홈으로 돌아가기]                      |
+--------------------------------------------------+
```

### 네트워크 오프라인

| 상태 | UI |
|------|-----|
| 오프라인 감지 | 상단 고정 배너: "인터넷 연결이 끊겼습니다. 연결을 확인해주세요." (orange) |
| 온라인 복구 | 배너 자동 해제 + toast("연결이 복구되었습니다") |

### 에러 로깅

- 클라이언트 에러: `console.error` + 선택적 외부 서비스 (Sentry 등)
- 서버 에러: Next.js 서버 로그
