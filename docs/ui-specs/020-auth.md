# Task 020: 로그인/회원가입 페이지

## 페이지 목적

시스템 사용자(운영자, 수사관, 관리자, 법무 담당)가 이메일/비밀번호 기반으로 로그인하고, 관리자 초대를 통해 회원가입하여 역할 기반 대시보드에 접근한다.

## 페이지 목록

| 라우트 | 설명 |
|--------|------|
| `/login` | 로그인 |
| `/signup` | 회원가입 (초대 기반) |

---

### 로그인 (`/login`)

#### 레이아웃

```
+------------------------------------------------------------------+
| (전체 화면, 중앙 정렬)                                            |
|                                                                    |
|              +-- 로그인 카드 (max-w-md) --------+                 |
|              |                                   |                 |
|              | [로고/서비스명]                   |                 |
|              | GambleGuard                       |                 |
|              | 불법 도박 사이트 자동 채증 시스템 |                 |
|              |                                   |                 |
|              | 이메일                            |                 |
|              | [admin@gambling-watch.kr_______]  |                 |
|              |                                   |                 |
|              | 비밀번호                          |                 |
|              | [●●●●●●●●____] [보기/숨기기]     |                 |
|              |                                   |                 |
|              | [ ] 로그인 상태 유지              |                 |
|              |                                   |                 |
|              | [        로그인        ]          |                 |
|              |                                   |                 |
|              | 비밀번호를 잊으셨나요? [재설정]   |                 |
|              +-----------------------------------+                 |
|                                                                    |
|              라이트/다크 모드 전환 토글                             |
+------------------------------------------------------------------+
```

#### 컴포넌트

| 컴포넌트 | 출처 | 용도 |
|----------|------|------|
| `Card`, `CardHeader`, `CardContent` | shadcn/ui | 로그인 폼 래퍼 |
| `Form`, `FormField`, `FormItem`, `FormLabel`, `FormMessage` | shadcn/ui | React Hook Form 통합 |
| `Input` | shadcn/ui | 이메일, 비밀번호 |
| `Button` | shadcn/ui | 로그인, 보기/숨기기 |
| `Checkbox` | shadcn/ui | 로그인 상태 유지 |
| `toast` (Sonner) | shadcn/ui | 로그인 성공/실패 |

#### 데이터

| 항목 | 값 |
|------|-----|
| API 엔드포인트 | `POST /api/auth/login` |
| 요청 타입 | `LoginRequest { email, password }` |
| 응답 타입 | `ApiResponse<LoginResponseData>` |
| 폼 스키마 | `loginSchema` (Zod: email 필수+이메일형식, password 필수+8자이상) |
| 인증 | NextAuth.js v5 Credentials Provider |

#### 인터랙션

| 동작 | 반응 |
|------|------|
| "로그인" 클릭 | Zod 유효성 검증 → Server Action (`signIn('credentials')`) → 성공: `/`(대시보드)로 리디렉트, 실패: 인라인 에러 |
| 비밀번호 "보기/숨기기" | Input type 토글 (password ↔ text) |
| "재설정" 클릭 | 비밀번호 재설정 이메일 발송 Dialog |
| 이미 인증된 상태로 접근 | `/`(대시보드)로 자동 리디렉트 |
| 5회 연속 실패 | "잠시 후 다시 시도해주세요" 메시지 + 30초 쿨다운 |

### 회원가입 (`/signup`)

#### 레이아웃

```
+------------------------------------------------------------------+
| (전체 화면, 중앙 정렬)                                            |
|                                                                    |
|              +-- 회원가입 카드 (max-w-md) ------+                 |
|              |                                   |                 |
|              | [로고/서비스명]                   |                 |
|              | GambleGuard 계정 생성             |                 |
|              |                                   |                 |
|              | 이름 *                            |                 |
|              | [홍길동________________________]  |                 |
|              |                                   |                 |
|              | 이메일 *                          |                 |
|              | [user@gambling-watch.kr________]  |                 |
|              |                                   |                 |
|              | 비밀번호 *                        |                 |
|              | [●●●●●●●●____]                   |                 |
|              | 강도: ████░ 강함                  |                 |
|              |                                   |                 |
|              | 비밀번호 확인 *                   |                 |
|              | [●●●●●●●●____]                   |                 |
|              |                                   |                 |
|              | 초대 코드 *                       |                 |
|              | [INVITE-ABC123________________]   |                 |
|              |                                   |                 |
|              | [       계정 생성       ]         |                 |
|              |                                   |                 |
|              | 이미 계정이 있으신가요? [로그인]  |                 |
|              +-----------------------------------+                 |
+------------------------------------------------------------------+
```

#### 컴포넌트

| 컴포넌트 | 출처 | 용도 |
|----------|------|------|
| `Card` | shadcn/ui | 회원가입 폼 래퍼 |
| `Form`, `FormField` | shadcn/ui | React Hook Form |
| `Input` | shadcn/ui | 이름, 이메일, 비밀번호, 초대 코드 |
| `Button` | shadcn/ui | 계정 생성 |
| 커스텀 비밀번호 강도 바 | 커스텀 | 5단계 강도 표시 (약함/보통/양호/강함/매우 강함) |
| `toast` (Sonner) | shadcn/ui | 생성 성공/실패 |

#### 데이터

| 항목 | 값 |
|------|-----|
| API 엔드포인트 | `POST /api/auth/signup` |
| 요청 타입 | `CreateUserRequest { email, password, name, role }` + `inviteCode` |
| 폼 스키마 | `createUserSchema` 확장 (비밀번호 확인 + 초대 코드 추가) |

#### 인터랙션

| 동작 | 반응 |
|------|------|
| "계정 생성" 클릭 | Zod 유효성 검증 → 초대 코드 검증 → 계정 생성 → toast("계정이 생성되었습니다") → `/login`으로 리디렉트 |
| 비밀번호 입력 | 실시간 강도 바 업데이트 |
| 비밀번호 확인 불일치 | 인라인 에러 "비밀번호가 일치하지 않습니다" |
| 초대 코드 무효 | 인라인 에러 "유효하지 않은 초대 코드입니다" |
| "로그인" 링크 클릭 | `/login`으로 이동 |
