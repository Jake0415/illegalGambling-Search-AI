# Task 016: AI 분류 검토 페이지

## 페이지 목적

AI(Claude Haiku, XGBoost, BERT, Ensemble)가 분류한 불법 도박 사이트의 결과를 수동으로 검토하여 승인/수정/반려한다. 특히 신뢰도 임계값(0.7) 미만인 저신뢰 분류 결과를 우선적으로 검토하고, 검토 결과를 학습 데이터로 활용하여 분류 모델의 정밀도를 향상시킨다.

## 페이지 목록

| 라우트 | 설명 |
|--------|------|
| `/review` | AI 분류 검토 큐 (테이블) |
| `/review/[id]` | AI 분류 상세 검토 (근거 확인, 승인/수정/반려) |

#### 데이터

| 항목 | 값 |
|------|-----|
| Mock 서비스 (상세) | `mockReviewService.getById(id)` |
| Mock 서비스 (승인) | `mockReviewService.approve(id, reviewerId)` |
| Mock 서비스 (반려) | `mockReviewService.reject(id, correctedCategory, reason, reviewerId)` |
| API 엔드포인트 (상세) | `GET /api/classification/:id` |
| API 엔드포인트 (승인) | `POST /api/classification/:id/approve` |
| API 엔드포인트 (반려) | `POST /api/classification/:id/reject` |
| 응답 타입 | `ApiResponse<ClassificationReviewItem>` |
| 폼 스키마 (반려) | `rejectClassificationSchema` (Zod: correctedCategory enum 필수, rejectionReason string 필수 max 500자) |

**ClassificationReviewItem 상세 표시:**

| 필드 | 위치 | 렌더링 |
|------|------|--------|
| `siteUrl` | 좌측 상단 | 텍스트 + 링크 → `/sites/[siteId]` |
| `screenshotUrl` | 좌측 중앙 | 이미지 뷰어 (클릭 시 풀사이즈 Dialog) |
| `aiResult.model` | 우측 | Badge (CLAUDE_HAIKU=blue 등) |
| `aiResult.category` | 우측 | 한글 카테고리명 |
| `aiResult.confidence` | 우측 | Progress 바 + 퍼센트 |
| `aiResult.evidence[]` | 우측 | 불릿 리스트 (type별 그루핑) |
| `reviewStatus` | 우측 상단 | StatusBadge |

**분류 근거 (evidence) 렌더링:**

| evidence.type | 표시 라벨 | 상세 내용 |
|---------------|-----------|-----------|
| `keyword_match` | 키워드 매칭 | keywords 배열을 불릿으로 나열, score 퍼센트 |
| `visual_analysis` | 시각 분석 | elements 배열을 불릿으로 나열, score 퍼센트 |
| `structural_analysis` | 구조 분석 | features 배열을 불릿으로 나열, score 퍼센트 |
| `content_analysis` | 콘텐츠 분석 | topics 배열을 불릿으로 나열, score 퍼센트 |
| `domain_pattern` | 도메인 패턴 | pattern 텍스트, score 퍼센트 |

#### 인터랙션

| 동작 | 반응 |
|------|------|
| "승인" 버튼 클릭 또는 `A` 키 | `mockReviewService.approve()` → toast("분류 결과가 승인되었습니다") → 자동으로 다음 대기 건으로 이동 |
| "수정" 버튼 클릭 또는 `E` 키 | 수정 Dialog 열림: 카테고리 Select + 변경 사유 Textarea (필수) → `mockReviewService.reject(correctedCategory, reason)` → toast + 다음 건 이동 |
| "반려" 버튼 클릭 또는 `R` 키 | 반려 Dialog 열림: 반려 사유 Textarea (필수) → `mockReviewService.reject(NON_GAMBLING, reason)` → toast + 다음 건 이동 |
| `→` (오른쪽 화살표) 키 | 다음 대기 건으로 이동 |
| "이전 건" / "다음 건" 버튼 | 인접 검토 항목으로 이동 |
| 스크린샷 클릭 | 풀사이즈 Dialog (원본 해상도) |
| 사이트 URL 클릭 | `/sites/[siteId]`로 새 탭 열기 |
| 수정 Dialog에서 사유 미입력 시 제출 | Zod 유효성 검증 실패 → FormMessage 에러 표시 |
| 검토 완료 후 다음 건 없음 | toast("모든 검토가 완료되었습니다") → `/review`로 리디렉트 |
