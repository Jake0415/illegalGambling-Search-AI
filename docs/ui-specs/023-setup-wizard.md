# Task 023: 초기 설정 위저드

## 페이지 목적

시스템 최초 설치 후 슈퍼어드민 계정을 생성하고, 데이터베이스/Redis 연결, 외부 서비스 API 키, 채증 파이프라인 설정을 단계별로 안내하여 시스템을 운영 가능한 상태로 구성한다.

## 페이지 목록

| 라우트 | 설명 |
|--------|------|
| `/setup` | 초기 설정 위저드 (5단계 스테퍼) |

## 현재 구현 상태

> 이미 부분 구현됨: `frontend/src/app/setup/page.tsx` + `frontend/src/components/setup/setup-wizard.tsx` + 5개 스텝 컴포넌트. 아래는 기존 구현을 기반으로 한 완전한 화면 정의서이다.

#### 컴포넌트 (기존 구현 기반)

| 컴포넌트 | 파일 | 용도 |
|----------|------|------|
| `SetupWizard` | `components/setup/setup-wizard.tsx` | 위저드 컨테이너 (상태, 스테퍼, 폼 관리) |
| `StepAccount` | `components/setup/steps/step-account.tsx` | 계정 설정 (이름, 이메일, 비밀번호 + 강도 바) |
| `StepDatabase` | `components/setup/steps/step-database.tsx` | DB 연결 (PostgreSQL, Redis + 연결 테스트) |
| `StepServices` | `components/setup/steps/step-services.tsx` | 외부 서비스 (API 키 5개 + 검증 버튼) |
| `StepConfig` | `components/setup/steps/step-config.tsx` | 시스템 설정 (브라우저, 프록시, 비용 한도) |
| `StepComplete` | `components/setup/steps/step-complete.tsx` | 설정 확인 요약 + 완료 화면 |
| `Card` | shadcn/ui | 폼 래퍼 |
| `FormProvider`, `useFormContext` | react-hook-form | 멀티스텝 폼 상태 |
| `Button` | shadcn/ui | 이전/다음/완료 |
| `Input` | shadcn/ui | 모든 입력 필드 |
| `Separator` | shadcn/ui | 섹션 구분 |
| `toast` (Sonner) | shadcn/ui | 연결 테스트, API 검증 결과 |

#### 데이터

| 항목 | 값 |
|------|-----|
| 폼 스키마 | `setupWizardSchema` (Zod: account + database + services + config) |
| 폼 타입 | `SetupWizardFormData` |
| API 엔드포인트 | `POST /api/setup` (완료 시 전체 설정 저장) |
| 연결 테스트 | `POST /api/setup/test-connection` (PostgreSQL, Redis) |
| API 검증 | `POST /api/setup/validate-key` (Claude, SMS, CAPTCHA, Google) |

**5단계 스텝 구성:**

| 스텝 | ID | 라벨 | 필수 입력 |
|------|-----|------|-----------|
| 1 | `account` | 계정 설정 | email, password, passwordConfirm, name, department, phone |
| 2 | `database` | 데이터베이스 | databaseUrl (postgresql:// 형식) |
| 3 | `services` | 외부 서비스 | claudeApiKey |
| 4 | `config` | 시스템 설정 | maxConcurrentBrowsers (1~20) |
| 5 | `complete` | 설정 확인 | (요약 표시 + 완료 버튼) |

#### 인터랙션

| 동작 | 반응 |
|------|------|
| "다음" 클릭 | 현재 스텝 유효성 검증 → 통과 시 다음 스텝, 실패 시 인라인 에러 |
| "이전" 클릭 | 이전 스텝으로 이동 (데이터 유지) |
| 스텝 인디케이터 번호 | 완료된 스텝은 primary 색상, 현재는 ring 강조, 미완료는 muted |
| 비밀번호 입력 | 5단계 강도 바 실시간 업데이트 |
| "연결 테스트" 클릭 | toast("연결 성공") 또는 toast.error("연결 실패") |
| "검증" 클릭 | toast("API 키가 유효합니다") 또는 에러 |
| "설정 완료" 클릭 | 전체 폼 제출 → Server Action → 성공: 완료 화면 표시 |
| "대시보드로 이동" 클릭 | `/`로 리디렉트 |
| 이미 설정 완료된 상태로 `/setup` 접근 | `/`로 자동 리디렉트 |
