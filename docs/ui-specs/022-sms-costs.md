# Task 022: SMS 비용 대시보드

## 페이지 목적

SMS 서비스별(PVAPins, GrizzlySMS, SMS-Activate) 사용량, 비용, 성공률을 실시간으로 모니터링하고, 일일/월간 사용 한도를 관리하며, 비용 추이를 차트로 시각화하여 예산 초과를 사전에 방지한다. 시스템 설정 페이지 내 위젯 또는 별도 탭으로 접근한다.

## 페이지 목록

| 라우트 | 설명 |
|--------|------|
| `/settings` (SMS 비용 탭) 또는 `/settings/sms-costs` | SMS 비용 대시보드 |

#### 컴포넌트

| 컴포넌트 | 출처 | 용도 |
|----------|------|------|
| `PageContainer` | 커스텀 공통 | 페이지 헤더 |
| `Card` | shadcn/ui | KPI 카드, 서비스별 상세, 한도 설정 |
| `Progress` | shadcn/ui | 한도 대비 사용률 바 |
| `Badge` | shadcn/ui | 서비스 상태 (정상/경고/오류/미설정) |
| Recharts `LineChart`, `Area` | recharts | 비용 추이 차트 |
| `ToggleGroup` | shadcn/ui | 기간 선택 (7일/30일/90일) |
| `Input` | shadcn/ui | 한도 설정 입력 |
| `Button` | shadcn/ui | 저장 |
| `Alert` | shadcn/ui | 한도 경고 배너 |
| `Tooltip` | shadcn/ui | 차트 호버 상세 |
| `toast` (Sonner) | shadcn/ui | 저장 결과 |

#### 데이터

| 항목 | 값 |
|------|-----|
| Mock 서비스 (비용 추이) | `mockAnalyticsService.getCostStats(filter)` |
| 응답 타입 | `ApiResponse<CostTrendItem[]>` |
| CostTrendItem | `{ date, smsCost, captchaCost, proxyCost, totalCost }` |
| 대시보드 요약 | `mockDashboardSummary.externalServices.smsProvider` |

#### 인터랙션

| 동작 | 반응 |
|------|------|
| 기간 토글 (7일/30일/90일) | 차트 데이터 범위 변경 |
| 차트 호버 | Tooltip으로 해당 날짜 상세 비용 표시 |
| 한도 설정 "저장" | ConfirmDialog → `mockSettingsService.update()` → toast |
| 한도 80% 도달 | KPI 카드 yellow 경고 + Alert 배너 |
| 한도 100% 도달 | KPI 카드 red 경고 + Alert 배너 ("SMS 발급이 중지되었습니다") |
| 서비스 잔액 $10 이하 | 해당 서비스 카드 yellow 경고 |
