# Task 019: 설정 페이지

## 페이지 목적

시스템 전반의 운영 설정(외부 서비스 API 키, 프록시, 채증 파이프라인 등)을 관리하고, 사용자 계정과 역할 기반 권한을 관리하며, 불법 도박 탐지용 키워드를 CRUD 관리하고, 시스템 전체 감사 로그를 조회하여 운영 투명성을 확보한다.

## 페이지 목록

| 라우트 | 설명 |
|--------|------|
| `/settings` | 시스템 설정 (탭 기반 설정 폼) |
| `/settings/users` | 사용자/역할 관리 |
| `/settings/keywords` | 키워드 관리 |
| `/settings/audit-log` | 감사 로그 뷰어 (시스템 전체) |

### 사용자 관리 (`/settings/users`)

> **운영 시나리오**: 역할은 슈퍼관리자(SUPER_ADMIN)와 관리자(ADMIN) 2개만 존재한다. 사용자 추가/삭제는 슈퍼관리자만 가능하고, 나머지 기능 권한은 동일하다. 슈퍼관리자는 모든 사용자 정보를 수정할 수 있고, 관리자는 자신의 정보만 수정 가능하다. 최초 접속 시 셋업 위저드에서 슈퍼관리자 계정을 생성한다.

### 키워드 관리 (`/settings/keywords`)

> **운영 시나리오**: 운영자가 불법 사이트에 자주 사용되는 키워드를 주기적으로 등록하고, 시드 키워드를 기반으로 유사 키워드를 AI로 자동 생성한다. 각 키워드별 탐지 성과(탐지 사이트 수, 정밀도, 최근 추이)를 확인하며, 효과가 낮은 키워드는 비활성화하고 새로운 키워드를 추가하는 **주기적 관리 사이클**을 운영한다.

| 동작 | 반응 |
