# 3.1 탐지 엔진 기능 요구사항

## 개요

불법 도박 사이트 자동 탐지 엔진은 검색 기반 탐지, 크롤링 기반 탐지, AI 콘텐츠 분류, 도메인 호핑 추적, URL 패턴 분석, 스케줄링/큐 관리의 6개 하위 시스템으로 구성된다. 각 하위 시스템의 기능 요구사항을 FR-DE-001 ~ FR-DE-030 형식으로 정의한다.

### 비용 요약

| 항목 | 월 예상 비용 | 비고 |
|------|-------------|------|
| Google Custom Search API | ~$50 | 10,000 쿼리/월 기준 |
| Claude Haiku 4.5 (콘텐츠 분류) | ~$15 | 500사이트/월, 배치 API + 프롬프트 캐싱 적용 시 |

### 핵심 기술 스택

| 영역 | 채택 기술 | 근거 |
|------|----------|------|
| 크롤링 프레임워크 | Crawlee (PlaywrightCrawler) + rebrowser-playwright | Node.js 네이티브, Playwright 통합, 안티블로킹 내장 (decisions.md 확정) |
| HTTP 클라이언트 | got-scraping | TLS 핑거프린트 모방, Crawlee 생태계 (decisions.md 확정) |
| 핑거프린트 관리 | fingerprint-suite | Canvas/WebGL 핑거프린트 생성+주입 (decisions.md 확정) |
| AI 분류 | Claude Haiku 4.5 (few-shot) | F1 94-95%, 학습 데이터 불필요 (decisions.md 확정) |
| 작업 큐 | BullMQ + Redis | 크롤링 작업 스케줄링, 재시도, 우선순위 관리 |
| 검색 API | Google Custom Search API | 키워드 기반 불법 도박 사이트 탐색 |

## A. 검색 기반 탐지 (Phase 3)

| ID | 제목 | 설명 | 우선순위 | Phase | 수용 기준 | 기술 구현 참고 |
|----|------|------|---------|-------|----------|---------------|
| **FR-DE-001** | Google Custom Search API 연동 | Google Custom Search JSON API를 통해 불법 도박 관련 키워드로 웹 검색을 수행하고, 검색 결과(URL, 제목, 스니펫)를 구조화하여 수집한다. Programmable Search Engine을 생성하여 전체 웹을 대상으로 검색하며, API 키 및 검색 엔진 ID를 환경 변수로 관리한다. | P2 | Phase 3 | 1) API 호출 시 HTTP 200 응답과 함께 검색 결과 JSON을 정상 파싱한다. 2) 단일 쿼리당 최대 10건의 결과를 수집하며, `start` 파라미터로 페이징하여 최대 100건까지 확장 수집한다. 3) API 일일 할당량(10,000 쿼리/일) 소진 시 자동으로 요청을 중단하고 관리자에게 알림을 전송한다. 4) API 오류(429, 500 등) 발생 시 지수 백오프로 최대 3회 재시도한다. | Google Custom Search JSON API ($5/1,000쿼리), `googleapis` npm 패키지 |
| **FR-DE-002** | 불법 도박 키워드 세트 관리 | 불법 도박 사이트 탐지를 위한 키워드 세트를 DB에서 관리한다. 카테고리별로 키워드를 분류하고(스포츠 도박, 카지노, 경마 등), 관리자 대시보드에서 키워드 CRUD 기능을 제공한다. 초기 키워드 세트: 사설토토, 먹튀, 배당, 충전, 환전, 롤링, 꽁머니, 토토사이트, 사설카지노, 불법경마, 온라인슬롯, 바카라, 파워볼 등. | P2 | Phase 3 | 1) 최소 50개 이상의 초기 키워드가 카테고리별로 DB에 등록된다. 2) 관리자가 대시보드에서 키워드 추가/수정/삭제/활성화/비활성화를 수행할 수 있다. 3) 키워드별 탐지 건수 통계가 기록되어 효과 분석이 가능하다. 4) 키워드 변경 이력이 감사 로그에 기록된다. | PostgreSQL (키워드 테이블), Next.js 관리 UI |
| **FR-DE-003** | 키워드 조합 전략 | 단일 키워드, 복합 키워드(AND 조합), 제외 키워드(NOT 조합)를 활용한 검색 전략을 구현한다. 예: `"사설토토" AND "가입쿠폰"`, `"토토사이트" -"합법" -"스포츠토토공식"`. 키워드 조합 템플릿을 정의하고, 조합별 검색 효율(탐지율/오탐률)을 추적한다. | P2 | Phase 3 | 1) 단일 키워드, 2개 이상 AND 조합, NOT 제외 키워드 3가지 방식의 쿼리를 생성한다. 2) 최소 20개의 조합 템플릿이 사전 정의된다. 3) 각 조합별 검색 결과 중 실제 불법 사이트로 확인된 비율(정밀도)이 기록된다. 4) 정밀도 50% 미만인 조합은 자동으로 비활성화 대상으로 플래그된다. | Google Custom Search `exactTerms`, `excludeTerms` 파라미터 활용 |
| **FR-DE-004** | 검색 결과 중복 제거 및 필터링 | 검색 결과에서 동일 URL, 동일 도메인, 이미 수집된 사이트를 중복 제거한다. URL 정규화(프로토콜, www, 트레일링 슬래시 통일) 후 비교한다. 합법 사이트 화이트리스트(스포츠토토 공식, 한국마사회 공식 등)를 유지하여 오탐을 방지한다. | P2 | Phase 3 | 1) URL 정규화 후 동일 URL 중복이 100% 제거된다. 2) 화이트리스트에 등록된 도메인은 검색 결과에서 자동 제외된다. 3) 이미 DB에 등록된 사이트는 "기존 사이트" 태그가 부여되어 신규 탐지와 구분된다. 4) 필터링 후 남은 URL 목록이 탐지 파이프라인 큐에 자동 등록된다. | URL 정규화 라이브러리 (`normalize-url`), PostgreSQL UNIQUE 제약 |

## B. 크롤링 기반 탐지 (Phase 2-3)

| ID | 제목 | 설명 | 우선순위 | Phase | 수용 기준 | 기술 구현 참고 |
|----|------|------|---------|-------|----------|---------------|
| **FR-DE-005** | Crawlee PlaywrightCrawler 기반 크롤링 엔진 구축 | Crawlee의 `PlaywrightCrawler`를 사용하여 대규모 웹 크롤링 인프라를 구축한다. rebrowser-playwright를 Playwright 드롭인으로 사용하여 CDP 유출을 패치하고 안티봇 탐지를 우회한다(~85% 사이트 커버). 자동 스케일링으로 시스템 리소스에 따라 동시 크롤링 수를 동적 조절한다. | P0 | Phase 2 | 1) Crawlee PlaywrightCrawler가 rebrowser-playwright 기반으로 정상 초기화된다. 2) 동시 크롤링 수가 시스템 CPU/메모리 사용률에 따라 자동 조절된다 (최소 5, 최대 20 동시 세션). 3) 크롤링 세션당 메모리 사용량이 500MB를 초과하지 않는다. 4) Cloudflare JS Challenge가 적용된 사이트에서 자동 통과율 85% 이상을 달성한다. | Crawlee v3.16+ (`PlaywrightCrawler`), `rebrowser-playwright` (드롭인 교체), Crawlee `AutoscaledPool` |
| **FR-DE-006** | got-scraping HTTP 수준 TLS 핑거프린트 모방 | 경량 HTTP 요청(도메인 생존 체크, robots.txt 확인, 리디렉트 추적 등)에 `got-scraping`을 사용한다. 브라우저 수준 HTTP 헤더를 자동 생성하고, TLS ClientHello 핑거프린트를 실제 브라우저(Chrome/Firefox)와 동일하게 모방한다. Crawlee의 `HttpCrawler`와 통합하여 브라우저 렌더링이 불필요한 요청의 리소스를 절약한다. | P0 | Phase 2 | 1) got-scraping으로 전송한 HTTP 요청의 TLS 핑거프린트(JA3)가 실제 Chrome 브라우저와 동일하다. 2) HTTP 헤더 순서 및 값이 `header-generator` 패키지를 통해 자동 생성된다. 3) 프록시 설정이 URL 형식으로 간편하게 적용된다. 4) HTTP 수준 요청 처리량이 초당 50건 이상이다. | `got-scraping` npm 패키지, `header-generator` (Apify), Crawlee `HttpCrawler` |
| **FR-DE-007** | fingerprint-suite Canvas/WebGL 핑거프린트 주입 | Playwright 브라우저 인스턴스에 fingerprint-suite를 사용하여 고유한 브라우저 핑거프린트를 생성하고 주입한다. Canvas, WebGL, 폰트, 화면 해상도, navigator 속성 등을 실제 브라우저 프로필과 일치하도록 위장한다. 크롤링 세션마다 새로운 핑거프린트를 생성하여 세션 간 연관성을 차단한다. | P0 | Phase 2 | 1) `fingerprint-generator`로 생성한 핑거프린트가 실제 브라우저 프로필 데이터셋과 통계적으로 일치한다. 2) `fingerprint-injector`를 통해 Playwright 브라우저 컨텍스트에 핑거프린트가 정상 주입된다. 3) Canvas 핑거프린트 테스트 사이트(browserleaks.com 등)에서 각 세션이 고유한 핑거프린트를 표시한다. 4) 세션 간 핑거프린트 중복률이 1% 미만이다. | `fingerprint-suite` (Apify): `fingerprint-generator`, `fingerprint-injector`, rebrowser-playwright 조합 |
| **FR-DE-008** | 링크 분석 기반 관련 사이트 네트워크 추적 | 크롤링 대상 페이지에서 외부 링크를 추출하고, 링크 대상 사이트를 분석하여 관련 불법 도박 사이트 네트워크를 추적한다. BFS(너비 우선 탐색) 방식으로 최대 3단계(depth) 깊이까지 연관 사이트를 탐색한다. 발견된 관련 사이트를 그래프 형태로 관계를 시각화한다. | P2 | Phase 3 | 1) 단일 시드 URL에서 최대 depth 3까지 외부 링크를 추출하고 추적한다. 2) 추출된 링크 중 불법 도박 관련 사이트로 분류된 비율을 기록한다. 3) 사이트 간 관계가 방향 그래프(directed graph)로 DB에 저장된다. 4) 대시보드에서 네트워크 그래프 시각화가 가능하다. | Crawlee `RequestQueue` (BFS 큐), Crawlee `enqueueLinks()`, PostgreSQL (그래프 관계 테이블), D3.js 또는 Cytoscape.js (시각화) |
| **FR-DE-009** | 광고 네트워크 추적 | 크롤링 대상 페이지에서 광고 배너 이미지, 광고 스크립트(ad network 코드), 제휴 링크를 추출한다. 동일한 광고 배너나 광고 네트워크 코드를 공유하는 사이트들을 동일 운영자 그룹으로 클러스터링한다. 광고 이미지 해시 비교, 광고 스크립트 도메인 비교, 제휴 ID 추출을 수행한다. | P3 | Phase 3 | 1) 페이지 내 `<img>`, `<iframe>`, `<script>` 태그에서 광고 관련 요소를 추출한다. 2) 광고 이미지의 perceptual hash(pHash)를 생성하여 유사 이미지를 매칭한다 (해밍 거리 10 이내). 3) 동일 광고 네트워크를 공유하는 사이트를 2개 이상 발견하면 자동으로 클러스터를 생성한다. 4) 클러스터 정보가 대시보드에서 조회 가능하다. | `sharp` (이미지 처리), `imghash` (pHash 생성), 네트워크 요청 가로채기 (Playwright `route()`), Crawlee 요청 후킹 |

## C. AI 콘텐츠 분류 (Phase 1-2)

| ID | 제목 | 설명 | 우선순위 | Phase | 수용 기준 | 기술 구현 참고 |
|----|------|------|---------|-------|----------|---------------|
| **FR-DE-010** | Claude Haiku 4.5 few-shot 기반 도박 사이트 분류 | 크롤링된 웹페이지의 HTML 텍스트 콘텐츠를 Claude Haiku 4.5에 전송하여 불법 도박 사이트 여부를 분류한다. 시스템 프롬프트에 도박 사이트 분류 전문가 역할을 부여하고, 한국어 도박 용어 사전(배팅, 충전, 환전, 롤링, 꽁머니, 먹튀 등)과 5~10개의 few-shot 예시를 포함한다. 프롬프트 캐싱을 적용하여 반복 시스템 프롬프트의 비용을 90% 절감한다. | P0 | Phase 1 | 1) 불법 도박 사이트 분류 F1 스코어가 94% 이상을 달성한다 (수동 레이블링 100건 기준 검증). 2) 분류 응답 시간이 평균 3초 이내이다. 3) 프롬프트 캐싱 적용 시 입력 토큰 비용이 캐싱 미적용 대비 80% 이상 절감된다. 4) API 오류 발생 시 자동 재시도(최대 3회)가 동작한다. | Claude Haiku 4.5 API (`@anthropic-ai/sdk`), 프롬프트 캐싱 (`cache_control: {"type": "ephemeral"}`), 배치 API (50% 할인) |
| **FR-DE-011** | 분류 카테고리 체계 정의 | 웹사이트를 다음 5개 카테고리로 분류한다: (1) 불법 스포츠 도박 (사설토토, 배팅사이트), (2) 불법 경마 (사설경마, 온라인경마), (3) 카지노 (온라인카지노, 바카라, 슬롯), (4) 기타 도박 (파워볼, 사다리, 로또 조작 등), (5) 비도박 (정상 사이트). 각 카테고리에 대한 분류 기준과 예시를 문서화한다. | P0 | Phase 1 | 1) 5개 카테고리가 DB 스키마와 API 응답에 enum으로 정의된다. 2) 각 카테고리별 분류 기준 문서가 작성되어 시스템 프롬프트에 포함된다. 3) 각 카테고리별 최소 2개 이상의 few-shot 예시가 준비된다. 4) 분류 결과가 카테고리 코드와 한국어 라벨 모두로 조회 가능하다. | Claude 시스템 프롬프트 설계, PostgreSQL enum 타입 |
| **FR-DE-012** | 신뢰도 점수 출력 | AI 분류 결과에 0~1 범위의 신뢰도 점수(confidence score)를 포함한다.
