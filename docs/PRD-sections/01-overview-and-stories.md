## 1. 제품 개요

### 1.1 제품명 및 비전

- **제품명**: illegalGambling-Search-AI (가칭: 도박사이트 자동 채증 시스템)
- **비전**: 인터넷상 불법 스포츠 도박 및 불법 경마 사이트를 AI 기반으로 자동 탐지하고, 법적 증거로 활용 가능한 3단계 스크린샷 채증을 자동 수행하는 통합 시스템을 구축하여, 수사기관의 불법 도박 단속 역량을 획기적으로 강화한다.

### 1.2 프로젝트 배경 및 필요성

#### 불법 도박 시장 현황

- 국내 불법 온라인 도박 시장 규모는 연간 수십조 원으로 추정되며, 매년 증가 추세
- 불법 도박 사이트는 도메인 호핑(빈번한 도메인 변경)으로 단속을 회피 — 실제 테스트에서 6개 사이트 중 4개가 접속 불가(도메인 변경)로 확인
- SMS OTP 기반 간이 인증, 팝업/광고 난립, Cloudflare 방어 등 기술적 장벽 존재
- 텔레그램, SNS, 커뮤니티 등을 통한 홍보 채널 다양화

#### 기존 수동 방식의 한계

| 한계 | 설명 |
|------|------|
| 인력 집약적 | 수사관이 직접 사이트에 접속하여 수동으로 스크린샷 캡처 |
| 증거 무결성 미보장 | 수동 캡처 시 해시/타임스탬프 미적용으로 증거 변조 가능성 |
| 확장성 부재 | 월 수백 건의 사이트를 개별 수동 처리 불가 |
| 도메인 호핑 대응 불가 | 실시간 도메인 변경 추적 메커니즘 부재 |
| 일회성 단속 | 지속적 모니터링 및 이력 관리 체계 부재 |
| 3단계 채증 어려움 | 회원가입(SMS OTP), 배팅 화면까지의 채증에 많은 시간과 노력 소요 |

### 1.3 타겟 사용자

| 기관 | 역할 | 주요 니즈 |
|------|------|----------|
| **사행산업통합감독위원회** | 불법 사행산업 감독 및 정책 수립 | 대규모 불법 사이트 현황 파악, 통계 리포트 |
| **경찰청 사이버수사국** | 불법 도박 사이트 수사 및 단속 | 법적 증거력 있는 채증 자료, 신속한 증거 확보 |
| **한국마사회 불법경마 감시팀** | 불법 경마 사이트 모니터링 | 불법 경마 사이트 자동 탐지, 지속적 모니터링 |
| **국민체육진흥공단** | 불법 스포츠 도박 감시 | 불법 스포츠 토토 사이트 탐지 및 채증 |
| **관련 수사 기관 및 감시 단체** | 현장 수사 지원 | 증거 패키지 생성, 법원 제출 자료 준비 |

### 1.4 핵심 가치 제안

| 구분 | 기존 방식 | 본 시스템 |
|------|----------|----------|
| **탐지** | 수동 검색 및 신고 의존 | AI 기반 자동 탐지 (Claude Haiku 분류, F1 94-95%) |
| **채증** | 수동 스크린샷 캡처 | rebrowser-playwright 기반 3단계 자동 채증 |
| **인증 우회** | 수사관 직접 SMS 인증 수행 | PVAPins Non-VoIP 자동 SMS OTP 인증 |
| **증거 무결성** | 관리 어려움 | SHA-256 해시 + OpenTimestamps + RFC 3161 이중 타임스탬프 자동 적용 |
| **처리 규모** | 인력 집약적 (월 수십 건) | 자동화로 대규모 병렬 처리 (월 500건 이상 목표) |
| **모니터링** | 일회성 단속 | 지속적 모니터링, 도메인 호핑 실시간 추적, 이력 관리 |
| **비용** | 수사관 인건비 중심 | 외부 서비스 $867~2,012/월 (SMS 인증 중심) |
| **증거 패키지** | 수동 문서 작성 | WARC + SingleFile + 스크린샷 + 메타데이터 자동 생성 |

### 1.5 프로젝트 범위

#### In-Scope (범위 내)

- 수동 URL 입력 및 자동 탐지를 통한 불법 도박 사이트 식별
- rebrowser-playwright 기반 3단계 자동 채증 (메인화면 → 회원가입/배팅시작 → 실제배팅)
- SMS OTP 자동 인증 (PVAPins Non-VoIP 1순위)
- 팝업/광고 자동 감지 및 처리
- CAPTCHA 자동 풀이 (CapSolver/2Captcha) 및 수동 개입 대시보드 (CDP Pause&Attach)
- SHA-256 해시 + OpenTimestamps + RFC 3161 기반 증거 무결성 관리
- Claude Haiku 4.5 기반 콘텐츠 분류 및 폼 필드 자동 탐지
- Crawlee 기반 대규모 크롤링 인프라
- 대시보드 (사이트 관리, 채증 현황, 통계/리포트)
- 법원 제출용 증거 패키지 자동 생성 (WARC + SingleFile + PDF 보고서)
- 레지덴셜 프록시 로테이션 (도메인 호핑 대응)

#### Out-of-Scope (범위 외)

- 실제 금전 거래 또는 입금 행위 (최소 금액 배팅 시뮬레이션까지만)
- 불법 도박 사이트 직접 운영 방해 또는 차단 조치
- 다크웹(.onion) 도박 사이트 탐지
- 해외 불법 도박 사이트 (국내 대상 한국어 사이트로 한정)
- 모바일 앱 기반 불법 도박 서비스 분석
- 법률 자문 또는 법적 판단 기능
- 실시간 영상 기반 도박 (라이브 카지노 스트리밍) 분석

## 2. 사용자 페르소나 & 스토리

### 2.0 시스템 운영 구조기획 → 아키텍처 설계 → 구현 → 테스트기획 → 아키텍처 설계 → 구현 → 테스트

```
┌─────────────────────────────────────────────────────────────┐
│                    자동 운영 파이프라인                        │
│                                                             │
│  운영자 검색 실행 → 유사 키워드 자동 생성 → 불법 사이트 탐지   │
│       → 사이트 정보 취합/적재 → 자동 채증 → 증거 패키지 생성   │
│                                                             │
│              ※ 시스템이 1차로 자동 구동                       │
└───────────────────────┬─────────────────────────────────────┘
                        │ 결과 전달
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
  ┌──────────┐   ┌──────────┐   ┌──────────┐
  │ 수사관    │   │ 관리자/   │   │ 법무/    │
  │          │   │ 감독관    │   │ 증거담당  │
  │ 채증 결과 │   │ 통계/현황 │   │ 증거 패키│
  │ 열람·활용 │   │ 모니터링  │   │ 지 검증  │
  └──────────┘   └──────────┘   └──────────┘
```

- **운영자**가 검색 키워드를 입력하면 시스템이 자동으로 유사 키워드를 생성하고, 불법 사이트를 탐지·취합·적재·채증까지 수행
- **수사관/관리자/법무 담당**은 시스템이 수집·가공한 정보를 전달받아 활용하는 역할

### 2.2 사용자 스토리

#### 페르소나 A: 운영자 (최운영) — 시스템 자동 구동 주체

**US-A1: 검색 실행 및 유사 키워드 자동 생성**

> As a 운영자, I want to 시드 키워드를 입력하면 시스템이 유사 키워드를 자동 생성하고 확장 검색을 실행하도록, so that 하나의 키워드로 다양한 변형·은어까지 포괄하는 대규모 탐지가 가능하다.

수용 기준:
- [ ] 시드 키워드(예: "사설토토") 입력 시 Claude Haiku가 유사 키워드를 10개 이상 자동 생성한다 (예: 토토사이트, 스포츠배팅, 먹튀검증, 배당사이트, 프로토 등)
- [ ] 생성된 키워드 목록을 운영자가 검토·수정·확정할 수 있다
- [ ] 확정된 키워드로 Google Custom Search API + 크롤링 탐지가 자동 실행된다
- [ ] 키워드 세트는 DB에 저장되어 향후 재사용·스케줄 실행에 활용 가능하다
- [ ] 키워드별 탐지 결과 건수가 실시간 표시된다

**US-A2: 불법 사이트 자동 취합 및 적재**

> As a 운영자, I want to 탐지된 불법 사이트 정보가 자동으로 취합·중복 제거·적재되도록, so that 별도 수작업 없이 향후 채증 및 분석에 활용할 수 있는 사이트 DB가 구축된다.

수용 기준:
- [ ] 탐지된 사이트 URL, 도메인, IP, WHOIS 정보가 자동 수집·적재된다
- [ ] URL 정규화 및 중복 제거가 자동 수행된다 (동일 사이트 재등록 방지)
- [ ] Claude Haiku 4.5 few-shot으로 불법 도박 여부가 자동 분류된다 (F1 94-95%)
- [ ] 분류 결과에 신뢰도 점수(0~1)가 부여되고, 저신뢰 항목은 검토 대기 상태로 표시된다
- [ ] 적재된 사이트는 자동 채증 큐에 등록되어 1~3단계 채증이 순차 진행된다
- [ ] 도메인 호핑 감지 시 기존 사이트와 자동 연결(클러스터링)된다

**US-A3: 자동 채증 파이프라인 실행 및 모니터링**

> As a 운영자, I want to 적재된 사이트에 대해 1~3단계 채증이 자동으로 실행되고, 파이프라인 상태를 모니터링하도록, so that 수백 건의 사이트를 병렬로 자동 처리할 수 있다.

수용 기준:
- [ ] 적재된 사이트에 대해 BullMQ 큐 기반으로 1→2→3단계 채증이 자동 순차 실행된다
- [ ] 동시 처리 수(동시 브라우저 인스턴스)를 설정할 수 있다
- [ ] 채증 진행 상황(대기/진행/성공/실패)이 실시간 대시보드에 표시된다
- [ ] 실패 건은 자동 재시도(최대 3회, 지수 백오프) 후 실패 큐로 이동한다
- [ ] SMS OTP/CAPTCHA 자동 처리 실패 시 수동 개입 큐에 등록되고 Slack 알림이 발송된다

**US-A4: 스케줄 기반 자동 실행**

### 3.6 비즈니스 가치 향상

**US-C5: ROI 자동 산출 대시보드**

수용 기준:
- [ ] 비용 항목별(SMS/프록시/CAPTCHA/LLM/서버) 실시간 집계가 표시된다
