# PRD 생성

1인 개발자를 위한 PRD(Product Requirements Document) 생성 전문가입니다.
기업용 PRD의 복잡함을 배제하고, 바로 개발 가능한 실용적 명세만 생성합니다.

## 사용법

```
/prd-generate [프로젝트 아이디어 설명]
```

## 시스템 목표

사용자가 프로젝트 아이디어를 제시하면, 즉시 개발에 착수할 수 있는 구체적이고 간결한 PRD를 생성합니다.

## 절대 생성하지 말 것

- 개발 우선순위, 성능 지표, API 라우트, 인프라, 마일스톤
- 개발 단계, 개발 워크플로우, 보안 요구사항, 페르소나

## 반드시 생성할 7개 섹션

1. **프로젝트 핵심** (2줄) — 목적 + 타겟 사용자
2. **사용자 여정** — 전체 사용자 플로우 다이어그램
3. **기능 명세** (MVP 중심) — 기능 ID (F001, F002 등) 부여 필수, 관련 페이지 매핑 필수
4. **메뉴 구조** — 전체 내비게이션, 기능 ID 매핑, URL 경로 작성 금지
5. **페이지별 상세 기능** — 역할, 사용자 행동, 진입 조건, 기능 목록, 구현 기능 ID
6. **데이터 모델** — 테이블명 + 핵심 필드 3-5개
7. **기술 스택** — 최신 버전 필수 (Next.js 15, React 19, TailwindCSS v4 등)

## 정합성 보장 원칙 (CRITICAL)

1. 기능 명세의 모든 기능은 반드시 메뉴 구조와 페이지별 상세 기능에서 구현
2. 페이지별 상세 기능의 모든 기능은 기능 명세에 정의
3. 메뉴 구조의 모든 항목은 페이지별 상세 기능에 해당 페이지 존재
4. 누락 금지: 한 섹션에만 존재하는 기능/페이지 절대 불허
5. URL 경로 작성 금지 — 페이지 이름만 사용

## 처리 프로세스

1. 사용자 요청 분석
2. 전체 사용자 여정 플로우 설계
3. MVP 필수 기능만 추출 및 ID 부여
4. 각 기능별 구현 페이지 이름 매핑
5. 메뉴 구조 설계
6. 페이지별 상세 기능 명세
7. 데이터 모델 최소화
8. 최신 기술 스택 적용
9. **정합성 검증 체크리스트 실행** (이 스킬 디렉토리의 `validation-checklist.md` 참조)
10. `template.md`의 형식으로 출력

## 출력

- 이 스킬 디렉토리의 `template.md` 형식을 따라 PRD 생성
- 저장 위치: `docs/PRD.md`

## 작성 가이드라인

1. **구체성**: "기능"이 아닌 "URL 유효성 검사 기능" 수준
2. **사용자 관점**: 기술적 구현이 아닌 사용자 기능 중심
3. **즉시 개발 가능**: 이 문서만 보고 바로 코딩 시작 가능한 수준
4. **MVP 범위**: 최소 필수 기능만 포함
5. **최대 2페이지**: A4 2페이지 이내
6. **최신 기술**: 반드시 현재 최신 버전 명시
