# 프로젝트 초기화

Next.js 스타터킷을 프로덕션 준비 개발 환경으로 체계적으로 초기화합니다.

## 사용법

```
/init-project
```

## 6단계 프로세스

### 1단계: 분석
- 전체 프로젝트 구조 매핑, 모든 컴포넌트 식별
- 파일을 필수/선택/제거 가능으로 분류
- 의존성과 사용법 문서화
- 데모/예제 콘텐츠 vs 핵심 기능 구별
- `docs/PRD.md` 있으면 프로젝트 요구사항 확인

### 2단계: 계획
- 제거할 파일/폴더 목록과 근거
- 정리가 필요한 코드 블록 식별
- 구조적 개선 계획
- 핵심 기능 변경 없음 보장

### 3단계: 실행
- 데모 페이지, 예제 컴포넌트, 샘플 데이터 제거
- 불필요한 API 라우트, 플레이스홀더 이미지 정리
- 과도한 주석과 보일러플레이트 제거
- 필수 설정 보존 (이 스킬 디렉토리의 `preserve-checklist.md` 참조)

### 4단계: 문서 업데이트
- `docs/PRD.md` 기반으로 README.md 재작성
- CLAUDE.md 상단에 프로젝트 설명 추가
- 이 스킬 디렉토리의 `cleanup-checklist.md`로 검증

### 5단계: 최적화
- import 문 최적화, 사용하지 않는 import 제거
- CSS 정리, 사용하지 않는 스타일 제거
- 설정 파일 최소화 검증

### 6단계: 검증
- `npm run build` 오류 없이 성공
- 깨진 import나 누락된 의존성 없음
- TypeScript 컴파일 성공
- README.md, CLAUDE.md 올바르게 업데이트됨

## 규칙

- 핵심 기능을 망가뜨리기보다 의심스러운 것은 보존
- 모든 주석은 한국어로 작성
- Chain of Thought: 분석 → 이유 → 계획 → 실행 → 검증 순서 준수
